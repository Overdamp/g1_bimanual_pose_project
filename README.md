# G1 Bimanual Pose Reaching Project (ปรับปรุง Coordinate Frame Goal Version)

โปรเจกต์นี้เป็นเวอร์ชันปรับปรุงสำหรับการฝึกฝนหุ่นยนต์ Unitree G1 (ร่างกายส่วนบนและแขนทั้งสองข้าง) เพื่อเคลื่อนที่เข้าหาเป้าหมายพิกัดอ้างอิงที่มีทั้งตำแหน่งและทิศทาง (Coordinate Frame / Pose Reaching) โดยมีการจำกัดความเร็วเพื่อความปลอดภัยและเพิ่มความเข้มงวดของระบบตรวจจับการชนตัวเอง (Self-Collision)

---

## 🌟 คุณสมบัติที่ปรับปรุง (Key Improvements)

### 1. เป้าหมายพิกัดอ้างอิง 6-DOF (XYZ Coordinate Frame Target - ขนาดเล็กและหมุนตามหุ่นยนต์)
* **การปรับขนาด**: ปรับขนาดแกนเป้าหมาย XYZ (`left_target`, `right_target`) ลงเหลือ `(0.10, 0.10, 0.10)` เพื่อให้มีขนาดเท่ากับแกนที่ติดตั้งไว้บนปลายข้อมือหุ่นยนต์ ทำให้สังเกตการทำแนวแกนประกบกันได้อย่างพอดี
* **การปรับมุมเริ่มต้น (Z-Axis Alignment)**: เนื่องจากตัวหุ่นยนต์ G1 สปอนเฉียงโดยหมุนรอบแกน Z ไป +90 องศา (`[0.7071, 0, 0, 0.7071]`) ตัวพิกัดเป้าหมายจำลองทั้งขวาและซ้ายจึงได้รับการปรับมุมเริ่มต้นหมุนรอบแกน Z ไป +90 องศาเพื่อสอดรับกับแนวตัวหุ่นยนต์โดยสมบูรณ์
* **ระบบสปอนแบบไต่ระดับการเรียนรู้ตามตำแหน่งมือหุ่นยนต์ (Hand-Aligned Curriculum)**: 
  * **ช่วงเริ่มต้นฝึกฝน**: เป้าหมายจะสปอนตรงกับพิกัดตำแหน่งและมุมทิศทางเริ่มต้นของปลายมือหุ่นยนต์ (ข้อมือซ้าย/ขวาในท่ายืนพร้อมทำงาน) โดยเพิ่มระยะเบี่ยงเบนตำแหน่งเพียงเล็กน้อย ($\pm 0.5$ ซม.) และแทบไม่มีการหมุนเอียง เพื่อให้ AI สามารถเรียนรู้ที่จะรักษาสมดุลมือให้อยู่บนเป้าหมายได้ทันทีตั้งแต่วินาทีแรก
  * **ช่วงกลาง-ท้ายฝึกฝน**: ขีดจำกัดระยะห่างเชิงเส้นจะค่อยๆ ขยายออกไปสูงสุด `30 ซม.` และมุมบิดเบี้ยว Euler จะขยายได้ถึง `180 องศา` (3.14 เรเดียน) แบบค่อยเป็นค่อยไปเมื่อจำนวนสเต็ปรวมในการรันระบบจำลองเพิ่มขึ้น (ไต่ระดับความยากสูงสุดที่ 150,000 สเต็ป)
  * **ควบคุมไม่ให้เกินพื้นที่โต๊ะ**: พิกัดเป้าหมายหลังสุ่มเบี่ยงเบนจะถูกทำการจำกัดขอบเขต (Clamping) ด้วยพิกัดสัมพัทธ์ของโต๊ะทำงานเสมอ เพื่อรับประกันว่าเป้าหมายจะไม่มีวันหลุดออกจากขอบเขตโต๊ะ และไม่เกินขอบเขตระยะเอื้อมของหุ่นยนต์ G1

### 2. การเพิ่มเป้าหมายและแกนอ้างอิงที่ปลายมือหุ่นยนต์ (End-Effector Coordinate Frame Markers)
* **การแสดงผลเชิงภาพ**: เพิ่มแกนอ้างอิง XYZ ขนาดเล็กลงที่ข้อต่อข้อมือซ้ายและขวาของหุ่นยนต์ G1 (`left_wrist_yaw_link` และ `right_wrist_yaw_link`) ผ่านการจัดโครงสร้าง USD แบบ Hierarchy (Child Prims)
* **การหมุนตามปลายแขน**: เมื่อหุ่นยนต์ยืดแขนหรือบิดข้อมือ ตัวแกนอ้างอิง XYZ ที่ปลายมือจะขยับและหมุนตัวสอดคล้องกันตามจริง ทำให้มองเห็นความแตกต่างระหว่างท่าทางปลายมือปัจจุบันกับเป้าหมายได้อย่างชัดเจน

### 3. การเพิ่มโต๊ะวางของด้านหน้าหุ่นยนต์และระบบป้องกันการชนโต๊ะ (Table Collisions)
* **การจัดวาง**: วางโต๊ะอเนกประสงค์ `packing_table.usd` ไว้ด้านหน้าตัวหุ่นยนต์ที่ตำแหน่งสัมพัทธ์ `pos=[0.0, 0.55, -0.3]` (หน้าโต๊ะสูงประมาณ Z=0.70 เมตร)
* **การป้องกันการชน**: เปิดใช้งานคุณสมบัติการชน (`collision_enabled=True`) บนตัวโต๊ะ และจำกัดเฉพาะส่วนแขน ลำตัว และเอวหุ่นยนต์ในการคำนวณเซ็นเซอร์แรงปะทะ หากส่วนของร่างกายหุ่นยนต์ขยับไปกระแทกกับโต๊ะหรือตัวหุ่นยนต์เอง จะถูกลงโทษด้วยคะแนนลบหนักสุด `-5.0` ในรางวัล `undesired_contacts`
* **ถอดกล่องสี่เหลี่ยมออกชั่วคราว**: นำกล่องสี่เหลี่ยมออกจากฉากจำลองชั่วคราวเพื่อลดความซับซ้อน ช่วยให้ AI โฟกัสไปที่การยื่นมือให้เข้าใกล้พิกัดและทิศทางของเป้าหมายให้สำเร็จเป็นหลักก่อน

### 4. โฟกัสเฉพาะแรงชนส่วนบน (Upper-body Collision Focus)
* **ปรับลดการรบกวน**: ถอดส่วนสะโพก (Hips) หัวเข่า (Knees) และข้อเท้า (Ankles) ออกจากระบบตรวจจับการชนสะสม เพื่อไม่ให้แรงยืนบดกับพื้นตามปกติมารบกวนการคำนวณการชน ช่วยเพิ่มเสถียรภาพในการฝึกฝน PPO

### 5. การควบคุมแบบสัมพัทธ์และความปลอดภัยในการเคลื่อนไหว (Relative Delta Control & Safety)
* **การปรับ Action Scale และควบคุมแบบสะสม**: เปลี่ยนมาใช้ระบบควบคุมตำแหน่งข้อต่อแบบสัมพัทธ์ (`RelativeJointPositionActionCfg`) แทนการคุมตำแหน่งสัมบูรณ์แบบเดิม โดยอัปเดตสเต็ปการขยับสะสมด้วยอัตราการเปลี่ยนขยับ (`scale=0.05` หรือประมาณ 2.8 องศาต่อวินาทีจำลองย่อย) 
* **แก้ปัญหาตัวหุ่นหยุดนิ่ง (Anti-Freeze)**: วิธีนี้ช่วยลดและแก้ปัญหาเดิมที่หุ่นยนต์ขยับแขนได้แค่นิดเดียวแล้วค้างหยุดนิ่งเนื่องจากโดนบล็อกขอบเขตมุมรอบจุดยืนเริ่มต้น โดยหุ่นยนต์สามารถสะสมแรงขยับข้อต่อแขนไปตามแนวแกนจนสุดระยะข้อต่อ เพื่อไขว่คว้าหาพิกัดเป้าหมายที่อยู่ห่างออกไปได้เต็มที่และราบรื่นอย่างปลอดภัยสูงสุด

## 🚀 วิธีการทดสอบและฝึกฝน (How to Run)

ก่อนเริ่มรัน ให้แน่ใจว่าระบบ GPU ทำงานเดี่ยวอย่างถูกต้องผ่าน `CUDA_VISIBLE_DEVICES=0` และเปิดใช้งานสภาพแวดล้อม Conda:

### 1. การทดสอบความถูกต้องอย่างรวดเร็ว (Dry-run Test)
รันจำลอง 2 สภาพแวดล้อม เป็นจำนวน 2 รอบฝึกในโหมดเบื้องหลัง (Headless) เพื่อเช็คการติดตั้งและ Syntax:
```bash
source /home/luke/miniconda3/etc/profile.d/conda.sh && conda activate env_isaaclab2026
CUDA_VISIBLE_DEVICES=0 python g1_bimanual_pose_project/train.py --num_envs 2 --max_iterations 2 --headless
```

### 2. การฝึกฝนแบบเต็มรูปแบบ (Full Training)
รันการฝึกซ้อมแบบคู่ขนาน 64 สภาพแวดล้อม และดูหน้าต่างจำลองทางสายตา (GUI) เพื่อสังเกตพิกัดเป้าหมาย XYZ และทิศทางของมือ:
```bash
source /home/luke/miniconda3/etc/profile.d/conda.sh && conda activate env_isaaclab2026
CUDA_VISIBLE_DEVICES=0 python g1_bimanual_pose_project/train.py --num_envs 64
```

โมเดลที่รันจะถูกเก็บบันทึกข้อมูล TensorBoard และไฟล์น้ำหนักโมเดล (.pt) ไว้ที่:
`logs/g1_bimanual_pose_reach/bimanual_pose_ppo/`

### 3. การกำหนด Seed ในการฝึกฝน
สามารถกำหนด Seed เพื่อให้ระบบจำลองและโครงข่ายประสาททำงานด้วยการสุ่มที่เหมือนเดิมทุกประการได้ผ่าน Option `--seed` (ค่าเริ่มต้นคือ 42):
```bash
CUDA_VISIBLE_DEVICES=0 python g1_bimanual_pose_project/train.py --num_envs 64 --seed 42
```

### 4. การเล่นและการประเมินผลนโยบาย (Policy Playback)
หลังจากเสร็จสิ้นการฝึกฝน สามารถโหลดโมเดลล่าสุดมาแสดงผลผ่านหน้าจอเพื่อประเมินประสิทธิภาพโดยระบุคำสั่งรัน:
* **รันโหลดโมเดลล่าสุดอัตโนมัติ**:
  ```bash
  CUDA_VISIBLE_DEVICES=0 python g1_bimanual_pose_project/play.py --num_envs 16
  ```
* **รันระบุโมเดล checkpoint ที่เจาะจง และระบุ Seed**:
  ```bash
  CUDA_VISIBLE_DEVICES=0 python g1_bimanual_pose_project/play.py --num_envs 16 --checkpoint logs/g1_bimanual_pose_reach/bimanual_pose_ppo/model_1500.pt --seed 42
  ```
